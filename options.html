Rule customization

Define how the extension cleans up specific sites. Your rules supplement the built-in ones — both apply when their URL patterns match.

How to write rules

What a rule does

When you click Extract page on a site that matches one or more rules, the extension takes a copy of the page, removes any elements matched by the rules' CSS selectors, converts the result to markdown, then removes any whole sections matching the rules' section headings.

Rules are removal-only. They cannot add content, click buttons, scroll the page, or change anything other than removing specific elements or sections. See ADR-0002 / ADR-0003 for the full contract.

URL pattern

A JavaScript regular expression matching the URL where the rule should fire. Examples:

What it matchesPattern
Any page on a domain^https://([^/]+\.)?example\.com/
One specific page^https://example\.com/specific/page/?$
Subdomains of a site^https://[^/]+\.example\.com/
Profile pages under a path^https://([^/]+\.)?example\.com/profile/

Need to test a pattern? Paste it into regex101.com and try it against the URL you care about. Set the flavor to JavaScript.

CSS selectors (cleanup)

One CSS selector per line. Every element matched by any selector is removed before the page is converted to markdown.

Finding a selector:

  1. Open the page in your browser.
  2. Right-click the element you want to remove → Inspect.
  3. In the dev tools, look at the highlighted element's tag, classes, and id.
  4. Write a selector using its class (.foo-bar), id (#foo), tag (nav), or attributes ([aria-label="Promoted"]).

Common patterns:

To targetSelector
An element by class.promoted-content
An element by id#cookie-banner
A specific tagnav, footer, aside
By attribute[aria-label*='Promoted' i]
By href substringa[href*='/messaging/compose']

Selectors that don't match anything are silent no-ops. Malformed selectors are ignored.

Section headings to remove

One heading per line, case-insensitive. After the page becomes markdown, any ## Heading matching one of these is removed along with everything under it (up to the next H1 or H2).

Use this when a section's HTML container has no stable class or id you can target with a CSS selector. For example, a profile-page rule might strip a noisy ## Activity or ## Interests section that the user doesn't want in their capture.

Tips

  • Save the rule, then go to the target page and re-extract to see the effect.
  • If the rule isn't doing anything, the selectors probably don't match. Right-click the noisy element and inspect again.
  • Prefer stable selectors (BEM-style block names like .scaffold-layout__aside, attributes like [role='complementary']) over utility classes like .css-1abc23d which change with every site deploy.
  • You can always toggle Verbose mode in the popup to skip all rules and see what's there raw — useful when refining selectors.
  • Export your rules occasionally as a backup. Storage is per-device.

Your rules

All rules are yours to edit and delete. On first install the extension seeds the default rules that ship with the extension. Use Restore defaults to re-add the shipped defaults if you've deleted them.
